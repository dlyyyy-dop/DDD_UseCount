# 部署步骤（在电脑上操作）

**第一步：注册 GitHub 账号** 打开 [github.com](https://www.github.com)，点「Sign up」注册，填邮箱、密码、用户名即可。

**第二步：新建仓库** 登录后点右上角「+」→「New repository」：

- Repository name 填：`ddd-system`
- 选「Public」
- 勾选「Add a README file」
- 点「Create repository」

**第三步：上传 HTML 文件** 进入刚建的仓库：

- 点「Add file」→「Upload files」
- 将你项目文件夹中的所有文件（包括 HTML、JS、JSON、图片）拖拽到上传区域。
- 把 `DDD_system.html` 文件名改为 `index.html`拖进去
- 点「Commit changes」

**第四步：开启 Pages**

- 点仓库顶部「Settings」
- 左侧菜单找「Pages」
- Source 选「Deploy from a branch」
- Branch 选择 `main` (或 `master`) 分支，并将文件夹设置为 `/(root)`，点「Save」

**第五步：等待2分钟** 刷新 Pages 页面，会出现一个网址，格式是：

```
个人
https://adamhtmei.github.io/YYYY_DDDSystem_WPA/DDD_system.html
组织
https://dlyyyy-dop.github.io/DDD/DDD_system.html
```

### 给 PWA 开发者的特别建议

- **HTTPS 强制性**：GitHub Pages 自动为你的网站提供 HTTPS 支持，这是 PWA 能够正常工作的先决条件。
- ==**缓存更新**：由于 Service Worker 会缓存文件，当你后续更新程序时，记得在 `sw.js` 中修改 `CACHE_NAME` 的版本号（例如从 `v1` 改为 `v2`），这样用户访问时才会强制更新最新的代码。==
- **测试入口**：部署成功后，用手机浏览器打开该链接，点击浏览器菜单中的“添加到主屏幕”，即可像原生 App 一样使用你的 DDD 系统。

# 功能设计亮点

### **功能1（药品查询）**

- 卡片式展示，每张卡片突出显示 DDD 值（大字橙色）
- 支持药品名 + 拼音首字母同时筛选，实时过滤
- 每张药品卡片底部新增绿色「➕ 加入使用强度计算」按钮
  - 点击后若计算列表中已存在同名同规格药品，弹出确认框提示选择「替换」或「保留已有」
  - 若不存在则直接加入并显示成功提示
  - 加入的药品会自动保存到 localStorage，刷新页面不丢失

### **功能2（使用强度计算）**

- 点击任意药品行展开，直接输入用药数量，实时预览 DDDs
- 已输入数量的行自动变为**橙黄色**并排在最上方
- AUD 计算结果显示在页面顶部的深色卡片中
- 筛选查找时**已输入数量保留不丢失**
- 删除药品
  - 每条药品行右侧红色 🗑 垃圾桶按钮（点击不会触发展开/折叠）
  - 点击后弹出确认框，确认后从计算列表移除该药品，同时清除其已输入的用药数量
  - 删除操作同步保存到 localStorage，刷新页面不丢失

# DDD数据源更新流程

### 操作方法

1. 在 `大连市友谊医院抗菌药物目录.xlsx` 中维护药品数据
2. 打开 HTML，点击 **「📂 导入数据」**
3. 选择更新后的 `大连市友谊医院抗菌药物目录.xlsx`
4. 点击 **「确认导入」**
5. 完成，数据自动保存 ✅

------

### 关键特性

| 特性         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **自动记忆** | 导入后数据存在浏览器本地，关闭再开自动加载，无需重复导入     |
| **来源标识** | 页面显示「内置示例数据」或「已导入自定义数据 · 导入时间」，一目了然 |
| **格式兼容** | 自动识别列名（含有无空格的情况），自动过滤空行和说明行       |
| **校验提示** | 文件格式错误或无有效数据时给出明确提示，不会误导入           |
| **默认保底** | 若从未导入过，自动使用内置的示例数据，程序不会空白           |

> **注意**：如果换了手机或换了浏览器，需要重新导入一次，因为数据存在当前浏览器里。
